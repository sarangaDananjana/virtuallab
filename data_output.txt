 Overview

Phase  1:  The  Steam  Fetcher  View  (Backend)  
Create  a  simple,  dedicated  Django  view  to  proxy  the  Steam  API.  This  avoids  CORS  issues  on  the  
frontend
 
and
 
keeps
 
your
 
server
 
IP
 
hidden.
 ●  Endpoint:  /admin/api/fetch-steam/<app_id>/  ●  Access:  @staff_member_required  ●  Logic:  1.  Receives  the  app_id.  2.  Makes  a  GET  request  to  
https://store.steampowered.com/api/appdetails?appids=<app_id>.
 3.  Parses  the  JSON  response.  4.  Returns  a  streamlined  JSON  object  to  the  frontend  containing:  ■  name  ■  detailed_description  (and  short_description)  ■  developers  (Array  of  strings)  ■  publishers  (Array  of  strings)  ■  genres  (Array  of  strings)  ■  pc_requirements  (Raw  HTML  strings  for  minimum/recommended)  ■  images:  A  combined  array  of  URLs  from  header_image,  capsule_image,  and  
screenshots.
 
Phase  2:  The  Side-by-Side  UI  Layout  (Frontend)  
Create  a  custom  Django  template  (e.g.,  admin/custom/steam_ingestion.html).  
Top  Bar  
●  App  ID  Input:  A  text  input  and  a  "Fetch  Game"  button.  ●  Action:  Triggers  an  AJAX/Fetch  call  to  the  Phase  1  endpoint.  
Left  Panel  (Steam  Data  Viewer)  
Read-only  display  populated  by  the  AJAX  response.  ●  Header:  Steam  Title.  ●  Text  Blocks:  Steam  Description  (rendered  HTML  so  you  can  read  it  easily).  ●  Tags:  Genres,  Developers,  and  Publishers  listed  as  plain  text  pills.  ●  System  Requirements:  The  raw  HTML  output  from  Steam,  rendered  visually  so  you  can

 ●  Image  Gallery:  A  CSS  grid  displaying  all  fetched  images.  ○  Interactive  Element:  Each  image  has  a  "Select"  checkbox  and  a  "Make  Primary"  radio  
button.
 
Right  Panel  (Authoritative  Django  Form)
